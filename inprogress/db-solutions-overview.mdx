So let's enrich ourselves with 10k feet overview of modern day storage solutions. This article is just a beginning!

## Other data processing conepts and tools

Once we listed some storage systems, let's see other related concepts.

## Table Formats / Lakehouse Metadata Layers

These provide ACID transactions, schema evolution, and time travel on top of data lakes (usually on Parquet/ORC files).

- Apache Iceberg – open table format for huge analytic datasets
- Delta Lake – from Databricks, now open source
- Apache Hudi – stream-processing–friendly table format
- Apache Drill – schema-on-read query layer

## Data Lakes & Lakehouses

Data lakes store raw, structured, semi-structured, and unstructured data at scale (often on cheap object storage). Lakehouses combine the flexibility of lakes with structured querying like warehouses.

- Apache Hadoop (HDFS-based data lake)
- AWS Lake Formation
- Azure Data Lake Storage (ADLS)
- Google Cloud Storage (as data lake foundation)
- Databricks Lakehouse Platform
- Snowflake (evolving into lakehouse capabilities)

## Big Data Processing Frameworks

These aren’t databases themselves, but they are storage+compute engines that make raw data usable.

- Apache Spark – distributed compute for batch & streaming
- Databricks – commercial platform around Spark & Delta Lake
- Apache Flink – stream-first distributed processing
- Presto / Trino – distributed SQL query engine
- Apache Hive – SQL on Hadoop
- Google Dataflow / Apache Beam – unified batch & stream pipelines
